# Replit Fix Packet: Home Screen Safe Area, Bottom Tab Cutoff, and Hero Portrait Cropping

## Goal

Fix three persistent UI problems on the Clinica Run/Home screen:

```text
1. Hero portrait is still too large and cropped.
2. “SPROUT HEALER” is too high and collides with the notch/dynamic island area.
3. Bottom navigation text, especially “PROFILE,” is being cut off at the lower right/bottom edge.
```

The current fix did not work because the hero image is still being rendered in a way that allows it to overflow or crop inside the home layout.

Do not rebuild the app.

Do not change battle mechanics.

Do not change hero data.

Focus only on the Run/Home screen layout, hero portrait display, and safe-area spacing.

---

# Main Required Fix

Do not use the full-body hero portrait as a background image behind the hero text.

Do not place the hero image inside the same horizontal banner as:

```text
Novice Guardian
Apprentice Stabilizer
XP bar
Tap to Change Hero
```

Instead, create a dedicated **Hero Portrait Stage** / **Character Display Frame**.

The home page should use this layout:

```text
Safe Area Top Padding

Header
Hero Portrait Stage
Hero Info Card
Main Action Button
Bottom Navigation with Safe Area Padding
```

The image must live inside the Hero Portrait Stage only.

# Root Cause to Fix

The portrait is likely still being displayed with one of these problems:

```text
resizeMode: cover
ImageBackground
absolute image positioning
overflow hidden on a small banner
negative margins
top/left anchoring
image larger than parent container
hero image placed behind text panel
```

Replace that behavior with:

```text
resizeMode: contain
centered Image component
fixed-height portrait stage
full image visible
no crop
no top-left anchor
```

# Safe Area Requirements

Use safe-area support on the main Run/Home screen.

The screen must respect:

```text
Top notch / dynamic island
Bottom home indicator
Bottom tab bar height
```

Use SafeAreaView or useSafeAreaInsets.

The home screen root should include top and bottom padding from safe area.

Example behavior:

```text
Header starts below the notch.
Bottom tab labels are fully visible.
PROFILE is not cut off.
No content overlaps the phone frame/home indicator.
```

If using `useSafeAreaInsets`, apply:

```text
paddingTop: insets.top + extra spacing
paddingBottom: insets.bottom + extra spacing
```

Use at least:

```text
top extra spacing: 12–20
bottom extra spacing: 10–20
```

# Hero Portrait Stage

Create a dedicated central image area.

This is the most important part.

Use a portrait container like:

```text
HeroPortraitStage
- width: 100%
- height: approximately 260–330 depending on screen height
- alignItems: center
- justifyContent: center
- overflow: hidden or visible depending on final look
```

Inside it:

```text
Image
- resizeMode: contain
- width: 90% of screen width
- height: 100% of portrait stage
```

Important:

```text
Do not let the image exceed the portrait stage height.
Do not let the image extend under the info card.
Do not let the image be cropped by a small banner.
Do not use cover.
```

If the image is still too large:

```text
Reduce image height.
Do not crop it.
```

If the character appears too small:

```text
Increase stage height.
Do not use cover.
```
