# Replit Fix Packet: Home Screen Hero Portrait Cropping + Gacha-Style Main Page Redesign

## Goal

Fix the home screen hero portrait issue and redesign the main Run/Home screen so it feels more like a gacha/RPG main lobby.

Current issue:

```text
The selected lead hero portrait appears zoomed/cropped into the top-left corner on the main page.
The full character is not visible.
The image appears inside a horizontal banner area instead of being presented as the central hero.
```

Desired result:

```text
The main page should show a large full-body hero image centered on the screen, similar to gacha game home screens.
The hero should not be cropped, zoomed, or anchored to the top-left.
The character should be the visual focus.
Event/feature icons should appear around the character.
Bottom navigation should remain.
```

Do not rebuild the whole app.

Do not change battle mechanics.

Do not change hero data, battle data, or progression logic.

Focus only on the main Run/Home screen layout and image rendering behavior.

---

# Likely Root Cause

The hero portrait is probably being displayed using one or more of these patterns:

```text
resizeMode: "cover"
ImageBackground
absolute positioning
overflow: hidden
fixed small banner container
negative margins or offsets
image width/height larger than container
top/left alignment instead of center alignment
```

This causes the full portrait to crop and zoom.

The fix is:

```text
Use Image instead of ImageBackground for the central hero.
Use resizeMode: "contain".
Center the image horizontally and vertically.
Use a large dedicated hero-stage area.
Do not place the full-body portrait inside a short horizontal banner.
Do not use cover for full-body character art.
```

# Hero Image Rendering Rules

For the selected lead hero image on the home screen:

```text
Use full-body portrait image.
Use resizeMode: contain.
Keep aspect ratio.
Center the image.
Do not crop.
Do not stretch.
Do not anchor to top-left.
Do not use the portrait as a banner background.
```

The image container should behave like a character stage:

```text
height: approximately 40–50% of screen height
width: 100%
alignItems: center
justifyContent: center
overflow: visible if possible
```

The image itself should behave like:

```text
maxHeight: 100%
maxWidth: 85–95% of screen width
resizeMode: contain
```

If the image still appears too small, increase the stage height instead of switching to cover.
