# Replit Instruction: Fix Mobile Battle Interface So Everything Fits on One Screen

## Goal

The current mobile battle UI is too tall and important information is hidden behind scrolling. The first screenshot shows text being cut off and the action panel blocking the enemy/clue area. The second screenshot is closer to the desired look, but I still want the player to see the full battle state without needing to scroll.

Fix the battle interface for mobile web in Replit.

The player should be able to see these on one screen:

```text
Enemy name
Enemy image
Disease system
Corruption bar
Stability bar
Codex or clue hint
Selected team
Action Points
End Turn
Tabs: Actions / Items / Call / Team
Available actions
```

The user should not need to scroll up just to see vitals, hidden clue area, or core battle information.

---

# Current Problems

Fix these issues:

```text
1. Enemy name wraps badly and gets cut off.
2. Top enemy section is too tall.
3. Action panel overlaps or blocks the Codex/clue section.
4. Hero cards are too large.
5. Action cards are too tall.
6. Text inside buttons is truncated too aggressively.
7. The screen requires scrolling to see important battle state.
8. The selected hero/team section takes too much vertical space.
```

---

# Main Layout Direction

Use a compact mobile battle layout.

The full battle screen should be divided into these compact zones:

```text
A. Enemy header: 20-25% screen height
B. Meters + clue/codex row: 15-20%
C. Team + AP row: 15-20%
D. Action area: remaining space
```

Do not let any single section dominate the screen.

---

# Fix 1: Compact Enemy Header

The enemy header should be horizontal and compact.

Layout:

```text
[Enemy Image] [Condition label + Enemy name + System badge]
```

Enemy image should be small:

```css
.enemy-portrait {
  width: 76px;
  height: 76px;
  flex-shrink: 0;
}
```

Enemy name should not become huge or wrap awkwardly.

Use:

```css
.enemy-title {
  font-size: clamp(1.6rem, 7vw, 2.4rem);
  line-height: 0.95;
  max-width: 100%;
  word-break: normal;
  overflow-wrap: anywhere;
}
```

But limit it to 2 lines if possible:

```css
.enemy-title {
  display: -webkit-box;
  -webkit-line-clamp: 2;
  -webkit-box-orient: vertical;
  overflow: hidden;
}
```

If the name is too long, reduce font size instead of letting it create a huge header.

For “Air Sprite Corruption,” it should display cleanly as:

```text
Air Sprite
Corruption
```

Not:

```text
Air Sprite
Corruptio
n
```

# Fix 5: Remove Blocking / Overlap From Action Panel

The action area should not be a large fixed overlay that covers the upper battle content.

Use one of these approaches:

## Preferred: Normal flex layout

Make the whole battle screen use CSS grid:

```css
.battle-page {
  height: 100dvh;
  display: grid;
  grid-template-rows:
    auto   /* enemy header */
    auto   /* meters */
    auto   /* codex/clues */
    auto   /* team/AP */
    1fr;   /* actions */
  overflow: hidden;
}
```

Then the action area gets remaining space:

```css
.action-panel {
  min-height: 0;
  overflow: hidden;
}
```

Do not use a huge `position: fixed` action panel unless absolutely necessary.

## If fixed action panel is currently used

Then reduce its height and add enough padding to the content, but do not let it cover the codex/team area.

```css
.action-panel {
  max-height: 42vh;
  overflow: hidden;
}
```

# Fix 10: Avoid Important Text Being Cut Off

For skill names, use truncation only after a reasonable width.

Bad:

```text
Breath of Da...
```

Better:

```text
Breath of Dawn
```

Use smaller text before truncating.

```css
.action-card-title {
  font-size: clamp(0.9rem, 3.7vw, 1rem);
}
```

For very long names like “Rapid Response,” show:

```text
Rapid Response
```

not:

```text
Rapid Respo...
```

Only truncate descriptions, not the core name if possible.

```css
  }

  .enemy-portrait {
    width: 72px;
    height: 72px;
  }

  .enemy-title {
    font-size: clamp(1.5rem, 6.8vw, 2.1rem);
    line-height: 0.95;
  }

  .codex-card {
    min-height: 40px;
    max-height: 44px;
  }

  .hero-card {
    min-height: 68px;
  }

  .action-card {
    min-height: 82px;
  }
}
```

Use `100dvh`, not only `100vh`, so mobile browser UI does not break the layout.
